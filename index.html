Security & Compliance — Specific Controls

Governance.
Specific controls, not talking points.

CareStack is HIPAA compliant at the PMS layer. That is not the same as governing AI agents that cross system boundaries. Here is the specific control map.

🔒 Runtime Policy Enforcement — Flex Gateway
Every agent call — CareStack, OpenClaw, Salesforce — passes through Flex Gateway before execution. Out-of-the-box A2A and MCP policies enforce authentication, rate limits, and data classification. This is runtime control, not just design-time configuration.
↳ Control point: between every agent and every system
🛡 PII Masking Before LLM Exposure
Patient names, DOBs, treatment codes, SSNs are redacted by Flex Gateway before being sent to any external LLM — OpenClaw, OpenAI, Anthropic, or otherwise. AI operates on de-identified data. HIPAA safe-harbor pattern — identical to how Wells Fargo eliminates screen-scraping.
↳ PHI never leaves your perimeter in identifiable form
👁 Full Audit Trail — Agent Visualizer
Real-time observability of every agent interaction — what was called, what data was accessed, which LLM responded, what action was taken. Exportable for HIPAA audit response and PE due diligence packages.
↳ Observable · traceable · exportable · PE-ready
🏥 Deployment — CloudHub or Runtime Fabric
Agent Fabric deploys in CloudHub (managed) or Runtime Fabric (near your agentic endpoints for low latency). Healthcare-specific data residency compliance supported. High availability with regional fault isolation.
↳ Your data stays where compliance requires it
